담당 기능은 다음과 같다.

* src/analysis.py에서 감정 분석 기능 구현
* 사용자가 작성한 일기 내용을 입력받아 감정 분석 수행
* 핵심 키워드 추출
* 사용자에게 따뜻한 한마디(comment) 생성
* Google Gemini API를 사용하여 분석 수행
* API 키가 없거나 오류가 발생하면 Mock 데이터를 반환하도록 구현
* 위기 상황을 감지하는 안전장치 구현

구체적인 요구사항은 다음과 같다.

* CRISIS_KEYWORDS 리스트 생성

* 다음과 같은 위기 키워드를 포함

  * "죽고 싶"
  * "살기 싫"
  * "자살"
  * "자해"
  * "사라지고 싶"

* detect_crisis(text) 함수를 구현하여 위기 키워드 포함 여부를 검사

* 위기 상황이 감지되면 분석 결과 대신 다음 정보를 반환

{
"crisis": true,
"message": "힘든 시간을 보내고 계신 것 같습니다. 자살예방상담전화 109 또는 정신건강상담전화 1577-0199를 이용해 보세요."
}

* analyze_entry(text) 함수를 구현

입력:

* 사용자가 작성한 일기 문자열(text)

출력 형식:

],

],
"comment": "사용자에게 전하는 따뜻한 한마디"
}

* Google Gemini API를 사용

* 모델은 gemini-2.5-flash-lite 사용

* 환경 변수 GEMINI_API_KEY에서 API 키를 읽어오기

* _mock_analysis(text) 함수를 작성하여 테스트용 더미 데이터를 반환

* Gemini 호출 실패 또는 예외 발생 시 _mock_analysis() 결과를 반환
